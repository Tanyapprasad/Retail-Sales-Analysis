# Retail-Sales-Analysis | SQL
An SQL project focused on data cleaning, handling null values, and analyzing retail sales data. Includes queries on customer trends, category performance, monthly sales, top customers, and time-based order analysis using CTEs and window functions.
# Tools & Technologies

  - PostgreSQL
  - SQL
  - pgAdmin 4

# Dataset

The dataset contains retail transaction details such as:

  - Transaction ID
  - Sale date & time
  - Customer details
  - Product category
  - Quantity, price, and total sales

# Objectives

  - Create and manage a retail sales database
  - Clean data by handling null values
  - Perform exploratory data analysis
  - Answer business-driven questions using SQL

# Key Analysis Performed

- Total sales and orders by category
- Unique customer count per category
- Monthly average sales and best-selling months
- Top 5 customers by total sales
- Sales distribution by time shifts (Morning, Afternoon, Evening)

# SQL Concepts Used

- GROUP BY, ORDER BY
- Aggregate functions (SUM, COUNT, AVG)
- CASE statements
- Common Table Expressions (CTEs
- Window functions (RANK())

# Findings

- Customers belong to different age groups with purchases across multiple product categories.
- Several transactions have a total sale value greater than 1000, indicating high-value purchases.
- Clothing and Beauty emerge as frequently purchased categories.
- Average sales vary across months, helping identify peak sales periods.
- A small set of customers contributes significantly to overall sales.
- Sales activity differs across morning, afternoon, and evening shifts.

# Conclusion

  This project demonstrates practical SQL skills used in real-world data analysis, including data cleaning, aggregation, and trend analysis.
 
  It helps understand customer behavior, sales performance, and business patterns.
